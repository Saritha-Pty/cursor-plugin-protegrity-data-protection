# Contributing to Protegrity Data Protection Plugin

We welcome contributions to make this plugin more powerful and useful!

## How to Contribute

### 1. Fork and Clone

```bash
git clone https://github.com/Saritha-Pty/cursor-plugin-protegrity-data-protection.git
cd cursor-plugin-protegrity-data-protection
```

### 2. Create a Feature Branch

```bash
git checkout -b feature/your-feature-name
```

### 3. Make Changes

- Follow the existing code structure and conventions
- Update documentation for any new features
- Add comments for complex logic
- Test your changes thoroughly

### 4. Commit and Push

```bash
git commit -m "Add your feature description"
git push origin feature/your-feature-name
```

### 5. Open a Pull Request

Create a PR with:
- Clear description of changes
- Reason for the changes
- Any breaking changes or deprecations
- Testing steps

## Contribution Areas

### Code
- Bug fixes and improvements
- New skills or commands
- Enhanced hooks and automation
- Performance optimizations
- Better error handling

### Documentation
- Usage examples
- API integration guides
- Troubleshooting tips
- Best practices
- Video tutorials

### Testing
- Integration tests
- Hook testing
- Edge case scenarios
- Cross-platform compatibility

### Ideas
- New entity types
- Custom detection rules
- Advanced analytics
- Additional integrations

## Code Style

- Use clear, descriptive naming
- Add comments for non-obvious logic
- Keep functions focused and small
- Follow shell script conventions (for hooks)
- Update related documentation

## Testing

1. Test locally before submitting
2. Verify all hooks work correctly
3. Test with different Protegrity configurations
4. Check error handling for edge cases

## Questions?

- Open an issue for questions or discussions
- Check existing issues for similar topics
- Reference the [Protegrity Documentation](https://developer.docs.protegrity.com/)

Thank you for helping make this plugin better! 🙏
